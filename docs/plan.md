# 台灣旅遊美食網站規劃

## 專案名稱

台灣散步筆記

## 專案定位

「台灣散步筆記」是一個以台灣各縣市為單位，整理景點與美食店家的旅遊資訊網站。

目前主要用途是方便自己記錄有興趣的景點與店家，但網站介面先以一般旅遊資訊網站的方式設計，保留未來開放給其他人使用的可能性。

第一版先完成「台南」內容，其他縣市先保留入口，之後再逐步擴充。

## 第一版目標

- 建立一個簡單、清楚、適合新手實作的旅遊資訊網站。
- 網站名稱使用「台灣散步筆記」，不只限定台南。
- 使用縣市分頁或導覽切換不同城市，例如台南、高雄、台中、台北、花蓮。
- 第一版只有台南有完整內容。
- 高雄、台中、台北、花蓮先顯示「即將新增」。
- 台南頁面以景點與實際美食店家為主要內容。
- 第一版不排行程，也不記錄「去過／想去」等個人狀態。
- 未來若加入個人收藏或造訪紀錄，應獨立成登入後的使用者功能，不直接混入公開地點資料。

## 建議檔案結構

```text
index.html
city.html
detail.html
style.css
script.js
data.js
images/
```

各檔案用途：

- `index.html`：首頁與縣市入口。
- `city.html`：城市地圖、篩選器與地點列表。
- `detail.html`：景點或店家簡易詳細頁。
- `style.css`：網站樣式與響應式設計。
- `script.js`：地圖、篩選、預覽卡片與頁面互動。
- `data.js`：景點與美食店家資料。
- `images/`：景點與店家圖片。

首頁主要讓使用者了解網站並選擇城市，可以包含：

## 城市頁整體呈現

城市頁採用「互動地圖＋卡片列表」的方式呈現。

以台南頁面為例：

```text
台南旅遊
探索台南的景點與美食店家

[全部] [景點] [美食]
[行政區] [分類] [特色標籤]

┌─────────────────────────────┐
│                             │
│          台南地圖           │
│                             │
│   藍色圓點：景點            │
│   橘色圓點：美食            │
│                             │
└─────────────────────────────┘

共收錄若干地點
景點數量｜美食店家數量

景點與美食卡片列表
```

地圖與下方卡片列表保留在同一頁：

- 地圖適合查看位置與附近地點。
- 卡片列表適合瀏覽、比較與閱讀資訊。
- 使用篩選條件後，地圖標記與卡片列表同步更新。

## 地圖功能

城市頁會顯示該縣市的互動地圖。

### 標記方式

- 景點使用一種顏色，例如藍色。
- 美食店家使用另一種顏色，例如橘色。
- 地圖上放置圖例，清楚說明不同顏色代表的類型。
- 未來也可以加入圖示，避免只依賴顏色辨識。

### 桌機版互動

- 滑鼠移到地圖圓點時，顯示小型預覽卡片。
- 點選圓點或預覽卡片後，進入對應的景點或店家詳細頁。

### 手機版互動

手機沒有滑鼠懸停，因此改為：

- 第一次點擊圓點，顯示預覽卡片。
- 再點擊「查看詳細資訊」，進入詳細頁。
- 預覽卡片可顯示於地圖下方或畫面底部，避免遮住過多地圖。

### 地圖與列表連動

未來可以加入：

- 滑鼠移到列表卡片時，對應地圖圓點突出顯示。
- 點擊「在地圖查看」時，地圖移動到該位置並開啟預覽卡片。
- 選擇行政區或分類後，地圖與列表同步篩選。

## 城市頁篩選功能

第一版可以先做簡單篩選：

### 第一層篩選

- 全部
- 景點
- 美食

### 進一步篩選

- 行政區
- 景點或料理分類
- 特色標籤

範例：

```text
行政區：全部｜中西區｜安平區｜東區

景點分類：
古蹟｜博物館｜自然景觀｜老街巷弄

美食分類：
牛肉湯｜台式小吃｜米食｜麵食｜甜點｜咖啡

特色標籤：
早餐｜宵夜｜老店｜可外帶｜免費景點｜雨天適合
```

第一版不需要一次放太多篩選條件，避免手機操作過於複雜。

## 台南美食內容

美食部分直接以「實際店家」為主要資料，不另外建立食物種類介紹頁。

店家賣什麼，透過料理分類、招牌品項與特色標籤說明。

例如：

```text
店家名稱：某牛肉湯店
行政區：安平區
料理分類：牛肉湯、台式小吃
招牌品項：溫體牛肉湯、牛肉燥飯
特色標籤：早餐、可外帶、老店
```

第一版可以先收錄約 6 間台南店家，種類可涵蓋：

- 牛肉湯
- 擔仔麵
- 碗粿
- 蝦捲
- 豆花
- 棺材板

## 分類與標籤規則

### 料理分類

說明店家主要賣什麼，例如：

- 牛肉湯
- 台式小吃
- 麵食
- 米食
- 甜點
- 冰品
- 飲料
- 咖啡

### 招牌品項

說明店內具體可以吃到什麼，例如：

- 溫體牛肉湯
- 牛肉燥飯
- 蝦仁飯
- 碗粿
- 豆花

### 特色標籤

說明店家的用餐特色或情境，例如：

- 早餐
- 宵夜
- 台南老店
- 銅板美食
- 可外帶
- 排隊名店
- 適合一人用餐
- 景點附近

分類、招牌品項與特色標籤應分開管理，方便未來篩選與搜尋。

## 經緯度取得方式

每個景點與店家都需要經緯度，才能顯示在地圖上。

第一版資料量不大時，建議：

1. 在 Google Maps 搜尋完整店名或景點名稱。
2. 確認地址與分店正確。
3. 在地圖位置按滑鼠右鍵。
4. 複製顯示的座標。
5. 以 `lat`、`lng` 寫入資料檔。
6. 實際在網站地圖上人工確認標記位置。

座標順序為：

```text
緯度 latitude, 經度 longitude
```

未來資料量增加時，可以再使用 Geocoding API 將地址批次轉成經緯度，但仍需要人工確認結果。

- 去過／想去／想再去等個人狀態。
- 個人筆記與造訪日期。
- 會員登入。
- 收藏功能。
- 評論與評分。
- 行程規劃。
- 後端資料庫。
- 即時營業狀態。
- 自動抓取 Google Maps 店家資料。
- 透過網站 UI 新增、刪除或修改景點與店家資料。
- 透過網站 UI 勾選、建立、重新命名或刪除分類與特色標籤。

## 未來可擴充功能

### UI 資料與標籤編輯功能（已記錄，暫不實作）

未來可加入管理者專用的編輯介面，讓管理者不需要直接修改 `data.js` 或 JSON 檔案，也能在網站上維護內容。

可能包含：

- 在景點或店家詳細頁點擊「編輯資料」。
- 從既有分類與特色標籤中勾選或移除項目。
- 修改名稱、介紹、地址、營業時間、招牌品項與經緯度。
- 新增景點或美食店家。
- 建立獨立的標籤管理頁，用來新增、重新命名、合併或刪除標籤。
- 一般訪客只能瀏覽，只有管理者可以修改資料。

若只做原型，可先使用 `localStorage` 搭配 JSON 匯入／匯出；若需要跨裝置同步、永久保存或未來公開使用，則再加入管理者登入、後端 API 與資料庫。

第一版仍維持手動修改 `data.js` 或 JSON 資料，不執行上述功能。

- 搜尋景點或店家。
- 更多行政區與標籤篩選。
- 收藏與個人清單。
- 登入後記錄去過、想去與個人筆記。
- 新增高雄、台中、台北、花蓮等縣市。
- 更完整的景點與店家詳細頁。
- 地圖標記群聚功能。
- 自動依目前地圖範圍篩選地點。
- 批次地址轉經緯度工具。
- 後端與資料庫。

## 目前確認的核心呈現

> 首頁讓使用者選擇城市；城市頁以互動地圖與卡片列表呈現景點和美食店家。景點與美食使用不同顏色標記，桌機滑鼠移到圓點時顯示預覽卡片，手機點擊圓點時顯示預覽卡片，再由卡片進入簡易詳細頁。第一版不顯示任何個人的去過或想去狀態。
