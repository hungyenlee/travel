# 地圖預覽卡自製，不使用 Leaflet 內建 popup

## 決策

城市頁（`city.js`）的地點預覽卡，採**自行掌控的浮動 / 底部卡片**，用原生 DOM 事件控制顯示與隱藏，**不使用 Leaflet 的 `bindPopup`／`openPopup`**。

- 桌機（≥768px）：滑鼠移入圓點 → 在圓點旁以 `position: fixed` 浮出預覽卡；移出圓點後延遲 250ms 關閉，期間若游標滑入卡片則取消關閉；點圓點直接進詳細頁。
- 手機（<768px）：點圓點 → 底部固定預覽卡。

## 為什麼記錄

未來看到 `city.js` 的人，很自然會想「Leaflet 本來就有 popup，為什麼要自己刻一個浮動卡片？」若不寫下來，很可能有人「順手」改回 `bindPopup`，結果把下面兩個已解決的問題又踩回去。

## 理由與取捨

1. **懸停 popup 的按鈕點不到**：原本用「滑入圓點開 popup、滑出圓點關 popup」。但使用者要按卡片上的「查看詳細資訊／查看店家資訊」按鈕時，游標必須離開圓點移到卡片上，這個「離開」瞬間就把卡片關掉了，按鈕永遠點不到（懸停式卡片的典型問題）。
2. **Leaflet popup 關閉行為不可靠**：實測中，連 Leaflet 內建的 `closeOnClick`／`closePopup` 在此情境都無法穩定關閉。

自製卡片改用：圓點 `mouseover` 顯示、`mouseout` 排定延遲關閉、卡片自身 `mouseenter` 取消關閉／`mouseleave` 關閉。如此游標可從圓點平順移到卡片、按到按鈕，離開卡片才收起。此行為以原生事件實作，可被確定地測試。

## 相關

- 預覽卡內容由 `shared.js` 的 `renderPreview()` 產生（景點與店家欄位不同）。
- 桌機懸停為滑鼠專屬行為；手機無懸停，改為點擊底部卡片。
