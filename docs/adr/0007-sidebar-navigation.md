# 全站左側側邊欄導覽；「我的行程」hover 浮出清單、plan 頁分總覽／規劃兩層

## 決策

把原本各頁寫死在右上角的頁首導覽（`台灣散步筆記 ＋ 探索城市 ＋ 我的行程`）改為**全站共用的左側側邊欄**；並把「我的行程」做成**桌機 hover 自動浮出行程清單、點擊進入總覽頁**，而 `plan.html` 分成「總覽」與「規劃」兩層。

要點：

1. **共用元件、動態渲染**：新增 `assets/js/sidebar.js`，在四頁的 `#app-sidebar` 內動態產生側邊欄，取代四份重複的 `<header>`。行程清單即時讀 `TripStore`。
2. **版面骨架**：四頁 `<body>` 改為 `.app-shell > (aside#app-sidebar + .app-content)`；`.app-content` 內維持原本的 `<main class="container">` 與頁尾。`<body data-page="explore|plan">` 標示當前頁高亮。
3. **「我的行程」＝連結＋hover flyout（無展開箭頭）**：
   - 桌機：滑鼠移到「我的行程」→ 右側自動浮出行程清單 flyout（**只顯名稱**）＋「＋新行程」。點某行程 → 進該行程規劃層（`plan.html?trip=<id>`）；點「＋新行程」→ 進總覽並自動開建立（`plan.html?new=1`）。
   - 點「我的行程」文字本身 → 進**總覽層**（`plan.html`）。
   - 手機：無 hover，flyout 不出現；點「我的行程」直接進總覽層。
4. **`plan.html` 兩層（依網址 `?trip`）**：
   - **總覽層**（無 `?trip`）：卡片列出所有行程，顯示「N 天 · M 個地點」＋「＋ 建立新行程」卡（就地命名）。是「我的行程」的落點。
   - **規劃層**（`?trip=<有效 id>`）：原本的天數／地圖／清單；頂部有「← 我的行程」返回總覽。`?trip` 指向不存在的行程時退回總覽。
   - 建立行程一律：命名 → `TripStore.createTrip` → 直接進該新行程的規劃層（`?trip=<新 id>`）。
5. **建立行程無死角**：不論從桌機 flyout、總覽卡、或空狀態，都能建立；總覽層即使一個行程都沒有，仍顯示「還沒有行程」＋「＋ 建立新行程」。
6. **手機版漢堡選單**：窄畫面（`max-width: 767px`）側邊欄收合，頂列只留漢堡鍵＋站名；點開由左側滑出抽屜＋半透明遮罩，點遮罩／✕／Esc 關閉。
7. **規劃層管理列簡化**（`plan.js`）：移除原本切換行程的 `<select>` 與「新行程」鈕；標題顯示當前行程名稱，旁邊保留「重新命名／刪除」（刪除後回總覽）。切換與建立行程改由側邊欄 flyout／總覽負責。
8. **「加入行程」面板不動**：城市頁／詳細頁的「＋ 加入行程」按鈕與其 popover（含裡面的建立新行程）維持不變，與側邊欄的建立入口互補。

## 為什麼記錄

這改動了兩條原本的基本決策：導覽原本刻意只留「首頁＋探索城市」（[decisions.md](../decisions.md) #18），而「我的行程」原是頂部導覽的一個連結、行程切換靠頁內 `<select>`（#27）。導覽是全站骨架，日後看程式的人會問「為什麼從頂部導覽改成側邊欄、為什麼行程用 hover 浮出、為什麼 plan 頁要分兩層」，把取捨記下來。

## 理由與取捨

1. **為什麼改側邊欄**：行程數量變多時，頂部一個 `<select>` 只能看到當前一個、切換要多一步；側邊欄能一眼列出全部行程並快速切換，也讓「建立／切換行程」在全站任何頁面都隨手可及。
2. **為什麼用 hover flyout 而非可展開下拉**：先前試過「箭頭展開的手風琴下拉」，但「文字是連結、只有箭頭能展開」造成同一列兩種行為、容易誤觸。改成 hover 浮出後，桌機「看一下有哪些行程」零點擊、要進頁面就點文字，行為單純；箭頭也可以拿掉，那一列回歸單純連結。
3. **為什麼 plan 頁分兩層**：「我的行程」需要一個「列出所有行程」的落點（總覽），而每個行程各自有天數／地圖／清單（規劃）。用網址 `?trip=<id>` 區分兩層，可分享／可重整回到同一狀態，也讓「返回總覽」就是一個單純連結。
4. **手機為什麼用「點文字進總覽」**：觸控沒有 hover，flyout 無法觸發；讓手機點「我的行程」直接進總覽頁（本來就列出所有行程），是 hover 版最自然的退化，不需要為手機另設一套展開互動。
5. **為什麼「加入行程」面板保留**：那是「邊逛景點邊順手收藏、可順帶開新行程」的主要情境，和側邊欄「純建立／切換」互補；移除會讓逛頁面時要收藏得先跳去側邊欄，體驗更差。
6. **為什麼用 JS 動態渲染而非四份靜態 HTML**：側邊欄含**會變動的行程清單**且四頁共用，靜態複製會四處不一致、且無法反映 localStorage。沿用專案「HTML 放插槽、JS 填內容」的既有精神（[ADR-0003](./0003-shared-layer-per-page-scripts.md)），不引入新相依（無框架、無 build）。
7. **flyout 溢出處理**：側邊欄改用 `overflow: visible` 讓 flyout 能向右浮出而不被裁切；flyout 以 `position: absolute; left: 100%` 貼在「我的行程」列右側，移入移出用小延遲（150ms）避免滑過空隙就收起。
