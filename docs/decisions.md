# 設計決策總表

這份文件整理「台灣散步筆記」第一版在規劃與實作過程中談定的決策，方便日後回顧。
用語定義見 [../CONTEXT.md](../CONTEXT.md)；較重大、需要理由的決策另有 ADR（見 [adr/](./adr/)）。

## 架構

1. **多頁式（MPA）**：三個實體 HTML 檔（`index.html`、`city.html`、`detail.html`），頁面間用網址參數傳遞。捨棄單頁式（SPA），因為對新手最直覺、也符合規劃書。
2. **英文 slug 當識別碼**：城市 `?city=tainan`、地點 `?id=chihkan-tower`。避免中文在網址被編碼成亂碼。城市另存「slug ↔ 中文名」對照。
3. **單一 `data.js`**：所有城市的地點放同一份清冊（`cities` + `places` 扁平陣列 + `taxonomy`），`place.city` 存 slug。維護時只需開一個檔。
4. **共用層 + 每頁一支 JS**：`shared.js` ＋ `home.js`／`city.js`／`detail.js`，取代單一 `script.js`。詳見 [ADR-0003](./adr/0003-shared-layer-per-page-scripts.md)。

## 分類與篩選

5. **篩選選項來自固定允許清單（taxonomy）**，非從資料自動推導；填了清單外用詞會顯示警告橫幅。詳見 [ADR-0001](./adr/0001-fixed-taxonomy-with-warning.md)。
6. **清單涵蓋四類**：行政區（每城市一份）、景點分類、料理分類、特色標籤。**招牌品項不納入清單、維持自由填寫**。
7. **篩選邏輯**：第一層（全部／景點／美食）單選；第二層（行政區／分類／標籤）**組內單選、組間 AND（交集）**。
8. **空結果**：顯示友善訊息「找不到符合條件的地點，試試調整篩選」+ 清除篩選按鈕；篩選列常駐「清除篩選」。
9. **清單外標籤的提示**：以**頁面橫幅**顯示，給維護者看的安全網（非給遊客）。

## 地圖

10. **Leaflet + OpenStreetMap**，免金鑰、免費、無帳單風險（付費服務如 Mapbox／Google 一律未使用）。
11. **初始視野自動框住（auto-fit）**所有（篩選後的）地點座標，`cities` 不需存中心點／縮放。
12. **標記**：景點藍點、美食橘點（顏色由 CSS 變數提供，與圖例一致）＋ 圖例。
13. **預覽卡互動**：桌機滑鼠移入圓點浮出預覽卡、可移到卡片按按鈕、點圓點進詳細頁；手機點圓點顯示底部固定卡。改為自製卡片、不用 Leaflet popup，詳見 [ADR-0002](./adr/0002-self-managed-map-preview.md)。
14. **地圖／列表／摘要同步**：套用任何篩選時，三者用同一份結果一起重繪。

## 圖片與連結

15. **圖片**：真圖與佔位圖混用；缺圖或檔案不存在時自動退回 `assets/images/placeholder.svg`；卡片圖統一裁成 4:3。
16. **詳細頁大圖可點 → 開 Google 地圖（新分頁）**；列表卡、預覽卡的圖維持「進詳細頁」。`mapUrl` 直接存維護者提供的 Google 地圖連結。

## 頁面與內容

17. **首頁精選隨機**：每次載入從台南隨機抽景點 3、店家 3，不在資料裡標記 featured。
18. **導覽只留「台灣散步筆記（首頁）＋探索城市」**，拿掉全站層級的「景點／美食」（第一版沒有跨城市總表）。
19. **其他城市（高雄／台中／台北／花蓮）標「即將新增」、不可點**；把該城市 `available` 改為 `true` 並補地點即可開放。
20. **詳細頁版型**：桌機左圖右資訊、手機由上而下；景點與店家共用同一頁，僅少數欄位（招牌品項、開放／營業時間標籤）不同。

## 內容現況（第一版）

- 新增「赤崁中藥行」時，把「**酒吧**」設為**料理分類**（而非特色標籤），以與「居酒屋、串燒」一致——描述店家「是什麼」的詞屬於料理分類。

## 實作期間的修正

- **`[hidden]` 保險規則**：`style.css` 補上 `[hidden] { display: none !important; }`。原因是 `.map-legend`、`.filter-bar` 等 class 上的 `display` 會蓋掉瀏覽器內建的 `[hidden]{display:none}`，導致這些元素在該隱藏時（例如「即將新增」城市頁）仍露出。
